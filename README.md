问题回答：

Blinn-Phong光照模型相较于Phong模型做了什么改进，两者的渲染结果是相同的吗。
  Blinn-Phong对Phong模型的改进主要在高光计算上。Phong模型是用反射向量R和视线向量V的点积来算高光，而Blinn-Phong引入了半角向量H，就是光线方向L和视线方向V的中间向量。这样就不用算反射向量R了，计算量小一点。但渲染结果不完全相同。Blinn-Phong的高光区域会稍微柔和、范围大一点，尤其在低光泽度时差别更明显。不过视觉上很接近，大多数情况看不出来。
  
简述一下什么是全局光照(Global Illumination)。
    全局光照就是模拟光线在场景里多次反弹的效果，不光是直接光照，还有物体间颜色的相互影响。比如红色墙壁会把周围染上红晕，角落不会太暗。但实时计算很贵，电影里常见，游戏里比较难。

简述一下什么是各向异性过滤，相较于Mipmap有什么优点以及使用Mipmap会增加多大的内存开销。
    Mipmap是预先存好几层缩小版纹理，但这样斜着看地面时，u和v方向缩放不一致，Mipmap只能按统一比例来，结果就糊了。各向异性过滤能按不同方向分别采样，斜视时纹理保持清晰。内存开销：Mipmap会增加约1/3的额外内存。因为每层缩小一半，总和比原图多33%。各向异性过滤在这个基础上还要更多。

纹理还能存什么
    纹理不只是存颜色。法线贴图存法线方向（看起来凹凸），粗糙度贴图控制表面光滑程度，金属度贴图标记哪里是金属，AO贴图存环境遮挡，高度贴图存海拔信息，甚至存发光色、透明度等。现代渲染里纹理就是数据容器。


完成任务的主要流程
一、使用几何辅助类创建几何体并贴上纹理
    我们用的是 Geometry::CreateSphere 这个辅助函数来生成球体网格。它接收半径、经线分段数和纬线分段数，返回一个包含顶点和索引的 mesh 对象。每个顶点包含了位置、法线和纹理坐标。拿到 mesh 之后，分别创建顶点缓冲区和索引缓冲区，把它们上传到显存里。
    纹理方面，我们没有用外部图片，而是在代码里程序化生成了一张 512x512 的彩色纹理。生成逻辑是遍历每个像素，把 UV 坐标转成极坐标，根据半径和角度计算条纹和圆环图案，混合出最终 RGB 颜色。这样生成出来的纹理看起来像彩色雷达图或者抽象星爆效果。然后用 CreateTexture2D 和 CreateShaderResourceView 把它绑定到着色器里。

二、使用 HLSL 编写 Blinn-Phong 光照模型
在像素着色器里，我们实现了标准的三部分光照计算：
    环境光：直接用环境光强度乘以材质环境光颜色，给暗部一个基础亮度，避免背面全黑。
    漫反射：计算光源方向与法线的点积，如果为正就保留，否则为0。再乘以光源颜色和材质漫反射颜色。这样就实现了面向光源的部分亮、背光部分暗的效果。
    高光：这是 Blinn-Phong 的核心。我们用视线方向和光源方向算出一个半角向量，再与法线点积，然后取若干次幂（shininess 控制）。这个计算比传统 Phong 模型更省指令，因为不需要算反射向量。高光会让物体表面看起来有金属或塑料的光泽感。
    最后把这三部分加起来，输出最终颜色。

三、让纹理旋转
    纹理旋转是通过在常量缓冲区里加了一个 texRot 矩阵实现的。在 CPU 端，我们每帧根据时间增量更新一个角度 sphereTexAngle，用这个角度的正弦和余弦构造一个 2D 旋转矩阵，然后转置后传到常量缓冲区。
    在像素着色器里，拿到这个旋转矩阵后，把它应用到采样用的 UV 坐标上。也就是说原本采样纹理用的是原始的 UV，现在先用旋转矩阵变换一下 UV 再做采样。这样纹理图案就会在球体表面慢慢转动，而球体本身的几何位置和法线都不变，看起来像是贴纸在球面上旋转。
    这个做法的好处是性能开销很小，不需要重新生成纹理或者修改顶点数据，只是每帧在采样前多了一步矩阵乘法。
